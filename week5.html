Week 5 Notes

  • There are 3 different types of errors. System error, Programmer error, and User error.
  • We can get refrence errors from small mistakes like not defineing a function. Here is some code to show this:

unicorn();

<< ReferenceError: unicorn is not defined

  • Here it shows that we need to define our function 'unicorn' in this case.
  • It is important to test and ebug because just one small mistake can disable a webiste or an important page of a website.
  • There is a built in protector for this called 'strict mode'. It can help us keep our code clean and with no errors
  • We would implement it like this:

'use strict'

  • Doing that will help us have another layer of protection with our code.
  • We can make new exceptions for code that might have a unique option.
  • We would do it like this:

throw 2;

throw 'Up';

throw { toys: 'out of pram' };

  • This helps us create and test our code for futher use and it can help us in the future for refrence.
  • We can test out code using jest functions by stating the method we want to test and the functions that corespond with it.
  • Here is an example:

function squareRoot(number) {

'use strict';

if (number < 0) {

throw new RangeError("You can't find the square root of negative numbers")

}

return Math.sqrt(number);

};

test('square root of 4 is 2', () => {

expect(squareRoot(4)).toBe(2);

});
